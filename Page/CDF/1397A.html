CodeForces 1397A

Juggling Letters

풀러 가기

번역

알파벳 소문자로 구성된 \( n \)개의 문자열 \( s_{1}, s_{2}, \ldots, s_{n} \)이 주어집니다.

1회의 행동마다 다음을 할 수 있습니다.
\( s_{i} \)에서 글자 하나를 지운 뒤, 그 글자를 \( s_{j} \)의 원하는 위치에 넣을 수 있습니다.
(\( j \)는 \( i \)와 같아도 됩니다.)

위 행동을 원하는 만큼 시행할 수 있습니다.
그렇다면, \( n \)개의 문자열을 모두 같게 만들 수 있나요?

힌트 0 (시간/공간 복잡도)

시간 복잡도는 \( O(N \times |S|) \)입니다.
공간 복잡도는 \( O(1) \)입니다.

힌트 1

글자를 몇 번 움직이든, 글자의 개수는 변하지 않습니다.

힌트 2

언제나 글자를 마음대로 움직일 수 있기 때문에,
글자의 위치는 상관이 없습니다.

힌트 3

글자의 위치는 상관이 없기 때문에, 사실상 신경써야 하는 건
"각 알파벳이 모든 문자열에서 동일한 횟수로 등장할 수 있는가?" 입니다.

풀이

"각 알파벳이 모든 문자열에서 동일한 횟수로 등장할 수 있는지"만 판별하면 됩니다.
개수만 맞다면, 각 글자를 언제든지 재배열해서 같은 문자열을 만들 수 있기 때문입니다.
그러기 위해서는, 각 알파벳이 \( n \)의 배수 회 등장해야 합니다.

\( n \)개의 문자열을 입력받으면서 각 알파벳의 등장 횟수를 세준 뒤,
각 알파벳의 등장 횟수를 \( n \)으로 나눈 나머지가 0인지 확인하면 됩니다.

코드 (C++)

#include <bits/stdc++.h>
#define FastIO ios_base::sync_with_stdio(0); cin.tie(0); cout.tie(0);
using namespace std;

void solve(){
    int n;
    cin >> n;
    int cnt[30] = {}; // 등장 횟수를 저장하는 배열
    for (int i = 1; i <= n; i++){
        string s;
        cin >> s;
        for (char c : s){ // 문자열의 각 글자마다
            cnt[c - 'a'] += 1;
        }
    }
    bool chk = true;
    for (int i = 0; i < 26; i++){
        if (cnt[i] % n != 0){ chk = false; } // 등장 횟수가 N의 배수인가?
    }
    if (chk){ cout << "YES" << endl; }
    else{ cout << "NO" << endl; }
}

int main(){
    FastIO;
    int t; cin >> t;
    while (t--){ solve(); }
}
