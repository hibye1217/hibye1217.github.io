leonld94' Fibonacci Function

함수

\[ F(n) = \sum_{ k=0 }^{ \lfloor (n-1)/2 \rfloor } \frac{ (n-k-1)! }{ k!(n-2k-1)! } \] \( F(n) \) returns \( n \)th Fibonacci Number
But... Why?

1. Simplify the function

let's replace \( n-k-1 \) to \( a \).
Then, our function \( F(n) \) will look like this.
\[ F(n) = \sum_{ k=0 }^{ \lfloor (n-1)/2 \rfloor } \frac{ a! }{ k!(a-k)! } \] And since \( \frac{ a! }{ k!(a-k)! } = _{a}C_{k} \), we can change \( \frac{ a! }{ k!(a-k)! } \) to \( { _{a}C_{k} } \).
\[ F(n) = \sum_{ k=0 }^{ \lfloor (n-1)/2 \rfloor } { _{a}C_{k} } \]

2. Analyze \( a \) and \( k \)

Let's write all possible \( (a, k) \) pair for some \( n \).
When \( n = 1 \), \( (0, 0) \) is possible.
When \( n = 2 \), \( (0, 1) \) is possible.
When \( n = 3 \), \( (0, 2), (1, 1) \) is possible.
When \( n = 4 \), \( (0, 3), (1, 2) \) is possible.
When \( n = 5 \), \( (0, 4), (1, 3), (2, 2) \) is possible.
When \( n = 6 \), \( (0, 5), (1, 4), (2, 3) \) is possible.
We can see 2 things here.
1. If \( k \) got bigger, then \( a \) got smaller.
2. If \( n \) is odd, then the number of possible combinations goes up by 1.

3. Pascal's Triangle

Pascal's Triangle looks like this.

\( 1 \)
\( 1 \) \( 1 \)
\( 1 \) \( 2 \) \( 1 \)
\( 1 \) \( 3 \) \( 3 \) \( 1 \)
\( 1 \) \( 4 \) \( 6 \) \( 4 \) \( 1 \)
Pascal's Triangle has 2 properties.
1. The position \( (n, r) \) has the value of \( { _{n}C_{r} } \)
2. When you draw a diagonal line, and add the numbers on that line, you get a Fibonacci Number.
For example, the Rednumbers sums up to 3 (4th Fibonacci Number).
And for Green numbers, it sums up to 5 (5th Fibonacci Number).

4. So...

When we analyzed \( a \) and \( k \), we got some \( (k, a) \) pairs.
When we draw that pair into the coordinate system, we have a diagonal line.
So that means, this function uses \( { _{a}C_{k} } \) to draw diagonal line at Pascal's Triangle, and sums it up to get Fibonacci Number.
So, It's all done. Except one thing...

5. 범위 확인하기

Since we don't know about range of this function, so let's check it.
When \( n \) is odd, the number of pairs goes up by 1.
So, if we adds 2 instead of 1, the number of pairs always goes up by 1.

When \( n = 1 \), The function passes \( (0, 0) \), and the diagonal line passes \( (0, 0) \).
When \( n = 2 \), The function passes \( (0, 1) \), and the diagonal line passes \( (0, 1) \).

If \( n \) satisfies the condition, then \( n+2 \) need to satisfy the condition as well.
If you think about it, the pairs of \( n+2 \) is
pairs of \( n \) then adds 2 to the \( a \), or adds 1 to both \( a \) and \( k \).
So, if the line's movement is same with range's change, we can say that this range satisfies the condition.
For first one, it's same. (Just move diagonal line down 2.)
So as the second one. (Just move diagonal line down 1, right 1.)

So that means, the diagonal line's range is same with the function's range.

Since the formula and range are both correct,
now we can say that this function calculates Fibonacci Number.
\( \mathbf{ Q.E.D. } \)
